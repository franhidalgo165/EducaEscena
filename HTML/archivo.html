Flujo de trabajo en Git

1. Pull (Descargar)

Antes de trabajar, siempre ejecuta Git: Pull. Esto trae a tu PC los cambios que otros (o tú desde otro equipo) hayan subido.

2. Trabajo

Realizas tus cambios en los archivos (como PROGRAMACION.PHP).

3. Commit (Guardar local)

En el icono de "Control de código fuente", pulsas + para preparar los archivos y luego escribes un mensaje para guardar el estado en tu PC.

4. Push (Subir)

Pulsas el botón de sincronizar (o Git: Push). Esto envía tus cambios a GitHub para que queden seguros y accesibles para otros.
